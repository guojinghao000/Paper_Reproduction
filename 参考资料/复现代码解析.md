### 问题分析

1. **依赖版本未锁定**：`requirements.txt` 中所有依赖均未标注版本号，这导致不同时间安装的依赖版本可能不同，严重影响可复现性。`timm`、`einops` 等库的 API 在不同版本间可能不兼容。

2. **PyTorch 版本差异**：论文使用 PyTorch 1.7.0，复现使用 1.13.1。虽然 API 基本兼容，但不同版本的算子实现（尤其是 CUDA 内核）可能导致数值差异，影响复现精度。

3. **缺少环境配置文件**：未提供 `environment.yml`、`Dockerfile` 或 `pip freeze` 输出，无法精确重建复现环境。

4. **无 Docker 镜像或一键安装脚本**：环境搭建需手动执行多步命令，增加了配置错误的可能性。

**结论：环境与依赖的可复现性较差，版本未锁定是最严重的问题。**

### 评价

**优点：**
- 模块划分基本合理：网络定义、数据加载、训练循环、配置管理各自独立
- `config.py` 使用 yacs 框架，配置管理较为规范，支持 YAML + CLI 覆盖

**问题：**

1. **冗余代码（已修复）**：~~`train.py:81-87` 和 `test.py:137-144` 中存在几乎完全相同的 `dataset_config` 字典构造逻辑，且硬编码覆盖了 CLI 传入的 `--list_dir` 参数。~~

2. **config.py 存在大量无关配置**：该文件源自 Swin Transformer 图像分类项目（Microsoft），保留了约 50% 的未使用配置项（如 Mixup、CutMix、AutoAugment、Color Jitter 等分类增强参数），增加了阅读负担。

3. **参数命名不一致**：`--n_class` 和 `--num_classes` 同时存在，通过 `dataset_config` 将 `args.n_class` 覆盖到 `args.num_classes`，逻辑迂回。

4. **trainer.py 中优化器硬编码**：SGD 优化器参数在 `trainer.py:50` 中直接写死，不接受 config 文件的 `TRAIN.OPTIMIZER` 配置——而配置文件默认是 AdamW。

5. **test.py 中有硬编码路径**：`test_log/` 路径为相对路径，且与 `model_out` 目录结构不统一。

**结论：代码结构基本合理，但存在配置冗余、硬编码和未清理的遗留代码。**

## 4. 模型实现

### 编码器

| 组件 | 论文描述 | 代码实现 | 一致性 |
|:-----|:--------|:--------|:------:|
| Patch Embedding | 4×4 不重叠分块，投影到 C=96 | `PatchEmbed` (Conv2d 4×4 stride 4) | ✅ |
| Stage 1 | 2× Swin Block, dim=96, heads=3 | `BasicLayer(depth=2)` | ✅ |
| Patch Merging 1 | 2× 下采样，dim → 192 | `PatchMerging` (4C→2C) | ✅ |
| Stage 2 | 2× Swin Block, dim=192, heads=6 | `BasicLayer(depth=2)` | ✅ |
| Patch Merging 2 | 2× 下采样，dim → 384 | `PatchMerging` | ✅ |
| Stage 3 | 2× Swin Block, dim=384, heads=12 | `BasicLayer(depth=2)` | ✅ |
| Patch Merging 3 | 2× 下采样，dim → 768 | `PatchMerging` | ✅ |
| Stage 4 | 2× Swin Block, dim=768, heads=24 | `BasicLayer(depth=2)` | ✅ |

### 瓶颈层

| 组件 | 论文描述 | 代码实现 | 一致性 |
|:-----|:--------|:--------|:------:|
| Bottleneck | 2× 连续 Swin Block, 分辨率不变 | 位于 `layers[3]` (第4个 BasicLayer 无 downsample) | ✅ |

### 解码器

| 组件 | 论文描述 | 代码实现 | 一致性 |
|:-----|:--------|:--------|:------:|
| Patch Expanding 1 | 2× 上采样，dim 768→384 | `PatchExpand` (Linear + rearrange) | ✅ |
| Decoder Stage 1 | Swin Block, dim=384, heads=12, skip concat | `BasicLayer_up(depth=2)` | ✅ |
| Patch Expanding 2 | 2× 上采样，dim 384→192 | `PatchExpand` | ✅ |
| Decoder Stage 2 | Swin Block, dim=192, heads=6, skip concat | `BasicLayer_up(depth=2)` | ✅ |
| Patch Expanding 3 | 2× 上采样，dim 192→96 | `PatchExpand` | ✅ |
| Decoder Stage 3 | Swin Block, dim=96, heads=3, skip concat | `BasicLayer_up(depth=2)` | ✅ |
| 最终上采样 | 4× 上采样 + 线性投影 | `FinalPatchExpand_X4` + Conv2d 1×1 | ✅ |

| 属性 | 论文 | 代码 | 一致性 |
|:-----|:-----|:-----|:------:|
| Swin Block 结构 | LN→W/SW-MSA→残差→LN→MLP→残差 | 同 | ✅ |

### 参数初始化

- 使用 `trunc_normal_(std=0.02)` 初始化线性层权重，偏置初始化为 0（`_init_weights` 方法）

**结论：模型结构高度对齐论文，核心组件（W-MSA/SW-MSA、Patch Merging/Expanding、跳跃连接）实现正确。主要细节偏差为解码器第4阶段仅 1 层（vs 编码器的 2 层）和单通道输入的特殊处理。**

## 7. 实验可复现性

### 随机种子设置

```python
random.seed(args.seed)          # 1234
np.random.seed(args.seed)
torch.manual_seed(args.seed)
torch.cuda.manual_seed(args.seed)
cudnn.benchmark = False
cudnn.deterministic = True
```

✅ 代码固定了 Python、NumPy、PyTorch、CUDA 的随机种子，并启用了 cudnn 确定性模式。

### 提供的运行脚本

| 脚本 | 用途 | 完整性 |
|:-----|:-----|:------:|
| `train.sh` | 训练启动 | ✅ 支持环境变量覆盖参数 |
| `test.sh` | 测试/推理 | ✅ |
| `generate_figures.sh` | 生成结果图 | ✅ |

### 训练资源估算

| 项目 | 数值 |
|:-----|:-----|
| GPU | NVIDIA RTX 4060 Laptop (8 GB) |
| 训练时间 (150 epochs) | ~21 小时 (实际日志时间跨度) |
| 每轮迭代 | 185 步 (训练) + 185 步 (验证) |
| 单轮耗时 | ~8 分钟 |
| 最低 GPU 要求 | 约 6 GB (batch_size=6 时) |

### 阻碍精确复现的因素

1. **依赖版本未锁定**（最关键因素）
2. **非确定性 CUDA 算子**：即使设置了 `cudnn.deterministic=True`，某些 scatter/gather 操作的梯度计算仍可能是非确定性的
3. **硬件差异**：不同 GPU 架构的浮点运算精度不完全一致
4. **验证集使用训练数据**：`best_model.pth` 的保存策略不可靠
5. **未提供训练好的模型权重**：无法直接验证推理结果

**结论：随机种子已固定，训练脚本完整。但依赖版本未锁定和硬件差异是复现的主要障碍。代码逻辑上的验证集问题进一步降低了复现的可信度。**
