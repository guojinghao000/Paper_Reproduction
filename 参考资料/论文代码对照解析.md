## 1. 编码器 (Encoder)

**对应论文：** 第 3.3 节 "编码器"

> 论文原文："在编码器中，分辨率为 H/4 × W/4 的 C 维标记化输入被送入两个连续的 Swin Transformer 块中进行表示学习……块合并层将减少标记的数量（2倍下采样）并将特征维度增加到原始维度的 2 倍。此过程将在编码器中重复三次。"

### 1.1 Patch Embedding（分块嵌入）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`PatchEmbed` 类 (第 534-579 行)

```python
class PatchEmbed(nn.Module):
    def __init__(self, img_size=224, patch_size=4, in_chans=3, embed_dim=96, norm_layer=None):
        # 使用 Conv2d 实现：kernel=4, stride=4
        self.proj = nn.Conv2d(in_chans, embed_dim, kernel_size=patch_size, stride=patch_size)
```

**功能说明：** 将输入图像 (224×224) 切分为 4×4 大小、互不重叠的图像块（patch）。每个 patch 的原始特征维度为 4×4×3=48，通过卷积投影到 96 维（`embed_dim=96`）。输出分辨率为 56×56（224/4），共 3136 个 token。对应论文 3.1 节中"医学图像被分割成大小为 4×4 的不重叠块，通过线性嵌入层将特征维度投影到 C"。

**调用位置：** `SwinTransformerSys.__init__()` 第 632-634 行
```python
self.patch_embed = PatchEmbed(img_size=img_size, patch_size=patch_size,
    in_chans=in_chans, embed_dim=embed_dim, norm_layer=norm_layer if self.patch_norm else None)
```

### 1.2 BasicLayer（编码器阶段）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`BasicLayer` 类 (第 405-472 行)

```python
class BasicLayer(nn.Module):
    def __init__(self, dim, input_resolution, depth, num_heads, window_size,
                 mlp_ratio=4., ..., downsample=None, ...):
        # 构建 depth 个 SwinTransformerBlock
        self.blocks = nn.ModuleList([
            SwinTransformerBlock(dim=dim, ...,
                shift_size=0 if (i % 2 == 0) else window_size // 2, ...)
            for i in range(depth)])

        # 下采样层：PatchMerging
        if downsample is not None:
            self.downsample = downsample(input_resolution, dim=dim, norm_layer=norm_layer)
```

**功能说明：** 编码器的每个阶段包含两个部分：
1. **偶数个 Swin Transformer Block**：第奇数个使用 W-MSA（窗口多头自注意力），第偶数个使用 SW-MSA（移位窗口多头自注意力），交替出现
2. **Patch Merging 层**（前 3 个阶段）：实现 2 倍下采样，同时将通道数翻倍

**前向传播**（第 453-461 行）：
```python
def forward(self, x):
    for blk in self.blocks:     # 依次通过所有 Swin Transformer Block
        x = blk(x)
    if self.downsample is not None:
        x = self.downsample(x)  # 最后通过 Patch Merging 下采样
    return x
```

### 1.3 编码器四阶段构建

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`SwinTransformerSys.__init__()` 第 650-665 行

```python
self.layers = nn.ModuleList()
for i_layer in range(self.num_layers):  # num_layers=4
    layer = BasicLayer(
        dim=int(embed_dim * 2 ** i_layer),               # 96→192→384→768
        input_resolution=(56//2^i, 56//2^i),             # 56→28→14→7
        depth=depths[i_layer],                            # [2,2,2,2]
        num_heads=num_heads[i_layer],                     # [3,6,12,24]
        window_size=window_size,                          # 7
        downsample=PatchMerging if (i_layer < 3) else None,  # 前3阶段有PatchMerging
        ...
    )
    self.layers.append(layer)
```

**功能说明：** 编码器共 4 个阶段，每阶段特征分辨率减半、通道数翻倍：

| 阶段 | 输入分辨率 | 通道数 | Swin Block 数 | 注意力头数 | 是否下采样 |
|:-----|:---------|:------|:------------|:--------|:---------|
| Stage 1 | 56×56 | 96 | 2 | 3 | PatchMerging → 28×28, 192 |
| Stage 2 | 28×28 | 192 | 2 | 6 | PatchMerging → 14×14, 384 |
| Stage 3 | 14×14 | 384 | 2 | 12 | PatchMerging → 7×7, 768 |
| Stage 4 | 7×7 | 768 | 2 | 24 | 无（进入瓶颈层） |

### 1.4 Patch Merging（块合并层，下采样）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`PatchMerging` 类 (第 303-349 行)

```python
class PatchMerging(nn.Module):
    def forward(self, x):
        H, W = self.input_resolution
        x = x.view(B, H, W, C)
        # 按照隔行隔列采样，分成 4 组
        x0 = x[:, 0::2, 0::2, :]  # 左上
        x1 = x[:, 1::2, 0::2, :]  # 左下
        x2 = x[:, 0::2, 1::2, :]  # 右上
        x3 = x[:, 1::2, 1::2, :]  # 右下
        x = torch.cat([x0, x1, x2, x3], -1)  # 拼接 → 4C
        x = self.norm(x)
        x = self.reduction(x)  # Linear(4C → 2C)
        return x
```

**功能说明：** 对应论文第 3.3 节中"输入块被分成 4 部分，并通过块合并层连接在一起。经过这样的处理，特征分辨率将下采样 2 倍。并且，由于拼接操作导致特征维度增加 4 倍，因此在拼接后的特征上应用线性层，将特征维度统一为原始维度的 2 倍。"

**具体操作：**
- 将特征图按 2×2 窗口切分为 4 组（左上、左下、右上、右下）
- 4 组拼接后通道变为 4C
- 经过 LayerNorm + Linear(4C → 2C)，输出通道为原来的 2 倍
- 分辨率从 H×W 降到 H/2 × W/2

## 2. Swin Transformer 块

**对应论文：** 第 3.2 节 "Swin Transformer 块"

> 论文原文："每个 Swin Transformer 块由层归一化（LN）层、多头自注意力模块、残差连接和带有 GELU 非线性的 2 层 MLP 组成。"

### 2.1 SwinTransformerBlock

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`SwinTransformerBlock` 类 (第 175-300 行)

```python
class SwinTransformerBlock(nn.Module):
    def __init__(self, dim, input_resolution, num_heads, window_size=7,
                 shift_size=0, mlp_ratio=4., ...):
        self.norm1 = norm_layer(dim)        # LayerNorm
        self.attn = WindowAttention(...)     # W-MSA 或 SW-MSA
        self.drop_path = DropPath(...)       # Stochastic Depth
        self.norm2 = norm_layer(dim)        # LayerNorm
        self.mlp = Mlp(...)                  # 2层 MLP, GELU 激活
```

**前向传播**（第 245-282 行）：
```python
def forward(self, x):
    # 公式(1): ẑ^l = W-MSA(LN(z^{l-1})) + z^{l-1}
    shortcut = x
    x = self.norm1(x)
    x = x.view(B, H, W, C)
    # 移位窗口：shift_size > 0 时先循环移位
    if self.shift_size > 0:
        shifted_x = torch.roll(x, shifts=(-self.shift_size, -self.shift_size), dims=(1, 2))
    # 窗口划分 + 自注意力
    x_windows = window_partition(shifted_x, self.window_size)
    attn_windows = self.attn(x_windows, mask=self.attn_mask)
    # 窗口还原 + 逆移位
    shifted_x = window_reverse(attn_windows, self.window_size, H, W)
    if self.shift_size > 0:
        x = torch.roll(shifted_x, shifts=(self.shift_size, self.shift_size), dims=(1, 2))
    x = x.view(B, H * W, C)
    x = shortcut + self.drop_path(x)         # 残差连接

    # 公式(2): z^l = MLP(LN(ẑ^l)) + ẑ^l
    x = x + self.drop_path(self.mlp(self.norm2(x)))
    return x
```

**功能说明：** 这是 Swin-Unet 最基本的计算单元。每个 Block 包含两个子层：
- **第一子层（自注意力）**：LayerNorm → 窗口/移位窗口多头自注意力 → 残差连接。对应论文公式 (1)(3)
- **第二子层（FFN）**：LayerNorm → 2 层 MLP（GELU 激活）→ 残差连接。对应论文公式 (2)(4)

**W-MSA 和 SW-MSA 的交替**：`shift_size` 决定当前 Block 使用哪种注意力。当 `shift_size=0` 时使用 W-MSA（常规窗口），当 `shift_size=window_size//2`（即 3）时使用 SW-MSA（移位窗口）。代码通过 `i % 2 == 0` 交替设置。

### 2.2 WindowAttention（窗口多头自注意力）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`WindowAttention` 类 (第 77-172 行)

```python
class WindowAttention(nn.Module):
    def __init__(self, dim, window_size, num_heads, qkv_bias=True, ...):
        self.relative_position_bias_table = nn.Parameter(
            torch.zeros((2*window_size-1) * (2*window_size-1), num_heads))
        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)  # QKV 投影
        self.proj = nn.Linear(dim, dim)                      # 输出投影
        self.softmax = nn.Softmax(dim=-1)
```

**前向传播**（第 125-156 行）：
```python
def forward(self, x, mask=None):
    # 1. 线性投影生成 Q, K, V
    qkv = self.qkv(x).reshape(B_, N, 3, num_heads, C//num_heads).permute(2,0,3,1,4)
    q, k, v = qkv[0], qkv[1], qkv[2]

    # 2. 计算注意力分数: Attention(Q,K,V) = SoftMax(QK^T/√d + B)V
    q = q * self.scale                       # 缩放因子 1/√d
    attn = (q @ k.transpose(-2, -1))         # QK^T

    # 3. 加上相对位置偏置 B
    relative_position_bias = self.relative_position_bias_table[
        self.relative_position_index.view(-1)].view(...)
    attn = attn + relative_position_bias.unsqueeze(0)

    # 4. 移位窗口的注意力掩码（防止跨窗口注意力）
    if mask is not None:
        attn = attn + mask

    # 5. SoftMax + Dropout
    attn = self.softmax(attn)
    attn = self.attn_drop(attn)

    # 6. 加权求和 + 输出投影
    x = (attn @ v).transpose(1, 2).reshape(B_, N, C)
    x = self.proj(x)
    return x
```

**功能说明：** 实现论文公式 (5) `Attention(Q,K,V) = SoftMax(QK^T/√d + B)V`。

- **相对位置偏置 B**：不同于 ViT 的绝对位置编码，Swin Transformer 使用相对位置偏置。`relative_position_bias_table` 是一个可学习的参数矩阵，存储了窗口内每对 token 之间的相对位置偏置。窗口大小为 7 时，相对位置范围为 [-6, 6]，共 13×13=169 种相对位置。
- **移位窗口掩码**：SW-MSA 中，由于窗口发生了移位，直接使用窗口划分会导致不同窗口的 token 错误地互相注意到。代码通过 `attn_mask` 将这些位置设为 -100（SoftMax 后趋近于 0）来隔离不同窗口。

### 2.3 MLP（多层感知机）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`Mlp` 类 (第 26-42 行)

```python
class Mlp(nn.Module):
    def __init__(self, in_features, hidden_features, act_layer=nn.GELU, drop=0.):
        self.fc1 = nn.Linear(in_features, hidden_features)  # 第一层：dim → 4*dim
        self.act = act_layer()                                # GELU 激活
        self.fc2 = nn.Linear(hidden_features, out_features)  # 第二层：4*dim → dim
        self.drop = nn.Dropout(drop)

    def forward(self, x):
        x = self.fc1(x)
        x = self.act(x)      # GELU 非线性
        x = self.drop(x)
        x = self.fc2(x)
        x = self.drop(x)
        return x
```

**功能说明：** 标准的 2 层全连接网络，中间使用 GELU 激活函数。隐藏层维度 = `mlp_ratio × dim`，默认为 4 倍（如 dim=96 时隐藏层为 384）。

### 2.4 窗口划分与还原

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，第 45-74 行

```python
def window_partition(x, window_size):
    # (B, H, W, C) → (B*num_windows, window_size, window_size, C)
    B, H, W, C = x.shape
    x = x.view(B, H//window_size, window_size, W//window_size, window_size, C)
    windows = x.permute(0,1,3,2,4,5).contiguous().view(-1, window_size, window_size, C)
    return windows

def window_reverse(windows, window_size, H, W):
    # (B*num_windows, window_size, window_size, C) → (B, H, W, C)
    B = int(windows.shape[0] / (H*W / window_size / window_size))
    x = windows.view(B, H//window_size, W//window_size, window_size, window_size, -1)
    x = x.permute(0,1,3,2,4,5).contiguous().view(B, H, W, -1)
    return x
```

**功能说明：** `window_partition` 将特征图切分为不重叠的窗口（如 7×7 大小），每个窗口内的 token 单独进行自注意力计算。`window_reverse` 将计算完的窗口还原为原始特征图形状。这是 Swin Transformer 计算效率的关键：将全局自注意力（复杂度 O(N²)）降低为窗口内自注意力（复杂度 O(M²×N/M²)=O(M²N)），其中 M 为窗口大小。

## 3. 瓶颈层 (Bottleneck)

**对应论文：** 第 3.4 节 "瓶颈层"

> 论文原文："由于 Transformer 太深难以收敛，瓶颈层仅使用两个连续的 Swin Transformer 块来构建，以学习深层特征表示。在瓶颈层中，特征维度和分辨率保持不变。"

**代码位置：** 编码器第 4 阶段 (`SwinTransformerSys.__init__()` 第 663 行)

```python
# i_layer = 3 (最后一个 encoder 阶段)
downsample = PatchMerging if (i_layer < self.num_layers - 1) else None  # 第4阶段无PatchMerging
# 即第4个 BasicLayer 的 downsample=None，只有2个 Swin Transformer Block
layer = BasicLayer(dim=768, input_resolution=(7,7), depth=2, ..., downsample=None)
```

**功能说明：** 瓶颈层就是编码器最后一个阶段（Stage 4），由 2 个连续的 Swin Transformer Block 组成，位于最小分辨率（7×7）和最大通道数（768）。与前面 3 个阶段不同的是，瓶颈层**没有 Patch Merging**，因此特征分辨率和维度保持不变。它承担"深度语义特征提取"的角色，是编码器和解码器之间的信息压缩点。

**为什么不是独立的 Bottleneck 模块？** 代码通过将第 4 个 `BasicLayer` 的 `downsample` 设为 `None` 来实现，而不是单独创建一个 Bottleneck 类。这比论文描述更简洁，但语义上等价。

## 4. 解码器 (Decoder)

**对应论文：** 第 3.5 节 "解码器"

> 论文原文："对称解码器基于 Swin Transformer 块构建……我们在解码器中使用块扩展层来上采样提取的深层特征。块扩展层将相邻维度的特征图重塑为更高分辨率的特征图（2倍上采样），并将特征维度相应地减少到原始维度的一半。"

### 4.1 BasicLayer_up（解码器阶段）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`BasicLayer_up` 类 (第 475-531 行)

```python
class BasicLayer_up(nn.Module):
    def __init__(self, dim, input_resolution, depth, num_heads, ..., upsample=None):
        # 与 BasicLayer 结构相同，使用 SwinTransformerBlock
        self.blocks = nn.ModuleList([...])
        # 上采样层：PatchExpand（代替 PatchMerging）
        if upsample is not None:
            self.upsample = PatchExpand(input_resolution, dim=dim, dim_scale=2, ...)

    def forward(self, x):
        for blk in self.blocks:
            x = blk(x)
        if self.upsample is not None:
            x = self.upsample(x)  # 通过 PatchExpand 上采样
        return x
```

**功能说明：** 解码器阶段与编码器阶段镜像对称，但有两个关键区别：
1. **下采样 → 上采样**：用 `PatchExpand` 替代 `PatchMerging`
2. **通道变化方向相反**：编码器通道数翻倍（dim→2dim），解码器通道数减半（dim→dim/2）

### 4.2 PatchExpand（块扩展层，上采样）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`PatchExpand` 类 (第 352-374 行)

```python
class PatchExpand(nn.Module):
    def __init__(self, input_resolution, dim, dim_scale=2, norm_layer=nn.LayerNorm):
        # 先 2 倍扩维：dim → 2*dim
        self.expand = nn.Linear(dim, 2*dim, bias=False)
        self.norm = norm_layer(dim // dim_scale)  # 上采样后的 LayerNorm

    def forward(self, x):
        x = self.expand(x)             # 维度翻倍: (B, H*W, C) → (B, H*W, 2C)
        x = x.view(B, H, W, 2*C)       # 还原为空间形状
        # Pixel Shuffle 重排: 2C → C//2, 宽高各翻 2 倍
        x = rearrange(x, 'b h w (p1 p2 c) -> b (h p1) (w p2) c', p1=2, p2=2, c=C//4)
        x = x.view(B, -1, C//4)        # 分辨率翻倍，维度降为 1/4
        x = self.norm(x)
        return x
```

**功能说明：** 这是 Swin-Unet 的核心创新之一，对应论文第 3.5 节的"块扩展层"。它与 Patch Merging 互为逆操作：

**以第一个 PatchExpand 为例（论文中的描述）：**
- 输入：`(B, 7×7, 8C)`，即 H/32 × W/32 × 8C
- Linear 扩维：`(B, 7×7, 16C)`
- Pixel Shuffle 重排：将每个 token 的 16C 通道中的空间维度信息"展开"
  - `p1=2, p2=2` 表示宽高各扩大 2 倍
  - 新通道数 `c = 16C / (2×2) = 4C`
- 输出：`(B, 14×14, 4C)`，即 H/16 × H/16 × 4C

与论文中描述完全一致："将输入特征的分辨率扩展到输入的 2 倍，并将特征维度减少到输入维度的四分之一"。

### 4.3 FinalPatchExpand_X4（最终 4 倍上采样）

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`FinalPatchExpand_X4` 类 (第 377-402 行)

```python
class FinalPatchExpand_X4(nn.Module):
    def __init__(self, input_resolution, dim, dim_scale=4, norm_layer=nn.LayerNorm):
        # 扩维 16 倍：dim → 16*dim
        self.expand = nn.Linear(dim, 16*dim, bias=False)
        self.output_dim = dim  # 输出通道为原始 dim
        self.norm = norm_layer(self.output_dim)

    def forward(self, x):
        x = self.expand(x)              # (B, H*W, dim) → (B, H*W, 16*dim)
        x = x.view(B, H, W, 16*dim)
        # 重排：宽高各扩大 4 倍，通道缩减为 1/16 = dim
        x = rearrange(x, 'b h w (p1 p2 c) -> b (h p1) (w p2) c',
                      p1=4, p2=4, c=C//(4*4))
        x = x.view(B, -1, self.output_dim)
        x = self.norm(x)
        return x
```

**功能说明：** 对应论文第 3.1 节中"使用最后一个块扩展层执行 4 倍上采样，将特征图的分辨率恢复到输入分辨率（W×H），然后对这些上采样特征应用线性投影层以输出像素级分割预测"。

**具体流程：**
- 输入：`(B, 56×56, 96)` — 解码器最后一层输出
- Linear 扩维 16 倍：`(B, 56×56, 1536)`
- Pixel Shuffle → `(B, 224×224, 96)` — 恢复为输入分辨率
- 最后接 `nn.Conv2d(96, num_classes, 1)` 输出分割预测

### 4.4 解码器四阶段构建

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`SwinTransformerSys.__init__()` 第 668-696 行

```python
self.layers_up = nn.ModuleList()
self.concat_back_dim = nn.ModuleList()
for i_layer in range(self.num_layers):

        layer_up = BasicLayer_up(dim=..., input_resolution=..., depth=...,
```

## 5. 跳跃连接 (Skip Connections)

**对应论文：** 第 3.6 节 "跳跃连接"

> 论文原文："与 U-Net 类似，跳跃连接用于将编码器的多尺度特征与上采样特征融合。我们将浅层特征和深层特征拼接在一起，以减少下采样造成的空间信息丢失。随后通过线性层，拼接后的特征维度保持与上采样特征的维度相同。"

### 5.1 编码器特征保存

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`SwinTransformerSys.forward_features()` 第 727-740 行

```python
def forward_features(self, x):
    x = self.patch_embed(x)       # Patch Embedding
    if self.ape:
        x = x + self.absolute_pos_embed
    x = self.pos_drop(x)
    x_downsample = []              # 存储各阶段输出的列表

    for layer in self.layers:
        x_downsample.append(x)     # 保存下采样前的特征 → 用于跳跃连接
        x = layer(x)

    x = self.norm(x)
    return x, x_downsample         # 返回瓶颈层特征 + 各阶段特征
```

**功能说明：** 前向传播过程中，`x_downsample` 列表保存了 4 个编码器阶段的**输入**特征（即每个 BasicLayer 处理前的特征图）。这些特征将在解码器中被复用。

### 5.2 跳跃连接融合

**文件：** `networks/swin_transformer_unet_skip_expand_decoder_sys.py`，`SwinTransformerSys.forward_up_features()` 第 743-754 行

```python
def forward_up_features(self, x, x_downsample):
    for inx, layer_up in enumerate(self.layers_up):
        if inx == 0:
            # 第一阶段：直接 PatchExpand，无跳跃连接
            x = layer_up(x)
        else:
            # 后续阶段：编码器特征与解码器特征拼接
            x = torch.cat([x, x_downsample[3 - inx]], -1)  # 通道维度拼接
            x = self.concat_back_dim[inx](x)                 # Linear 降维：2C→C
            x = layer_up(x)                                 # 通过解码器阶段
    x = self.norm_up(x)
    return x
```

**功能说明：** 跳跃连接的具体实现：
- `x_downsample[3-inx]`：取编码器对应分辨率的特征（镜像索引：inx=1→取 x_downsample[2]，即 1/8 尺度；inx=2→取 x_downsample[1]，即 1/4 尺度；inx=3→取 x_downsample[0]，即原始尺度）
- `torch.cat([x, x_downsample[3-inx]], -1)`：在通道维度拼接
- `concat_back_dim[inx]`：通过线性层将拼接后的 2C 降为 C

跳跃连接位于 3 个位置（1/4、1/8、1/16 分辨率尺度），对应论文表 4 中 `skip connection=3` 的设置。

```python
def load_from(self, config):

    pretrained_dict = torch.load(pretrained_path, map_location=device)
    pretrained_dict = pretrained_dict['model']

    model_dict = self.swin_unet.state_dict()

    # 忽略形状不匹配的参数
    for k in list(full_dict.keys()):
        if k in model_dict:
            if full_dict[k].shape != model_dict[k].shape:
                del full_dict[k]

    msg = self.swin_unet.load_state_dict(full_dict, strict=False)
```

**功能说明：** 这是论文强调的关键策略——"**镜像预训练初始化**"。

**核心逻辑：**
1. 加载 Swin-Tiny 在 ImageNet-22k 上的预训练权重
2. 编码器的 `layers.0`（浅层，56×56 分辨率）的权重 → 同时加载到解码器的 `layers_up.3`（浅层，56×56 分辨率）

### 8.2 DiceLoss 实现

**文件：** `utils.py` 第 9-45 行

```python
class DiceLoss(nn.Module):
    def __init__(self, n_classes):
        self.n_classes = n_classes

    def _one_hot_encoder(self, input_tensor):
        # 将标签转为 one-hot 编码
        tensor_list = []
        for i in range(self.n_classes):
            temp_prob = input_tensor == i
            tensor_list.append(temp_prob.unsqueeze(1))
        output_tensor = torch.cat(tensor_list, dim=1)
        return output_tensor.float()

    def _dice_loss(self, score, target):
        # Dice = 2|X∩Y| / (|X|+|Y|)
        # DiceLoss = 1 - Dice
        smooth = 1e-5
        intersect = torch.sum(score * target)
        y_sum = torch.sum(target * target)
        z_sum = torch.sum(score * score)
        loss = (2 * intersect + smooth) / (z_sum + y_sum + smooth)
        return 1 - loss

    def forward(self, inputs, target, weight=None, softmax=False):
        if softmax:
            inputs = torch.softmax(inputs, dim=1)    # 先 softmax
        target = self._one_hot_encoder(target)         # 标签转为 one-hot
        loss = 0.0
        for i in range(self.n_classes):
            dice = self._dice_loss(inputs[:, i], target[:, i])
            loss += dice                              # 各类别 Dice 损失相加
        return loss / self.n_classes                   # 取平均
```

**功能说明：** 多类别 Dice 损失。对每个类别分别计算 Dice 系数后取平均。核心公式 `Dice = 2|X∩Y| / (|X|+|Y|)`，损失为 `1 - Dice`。平滑因子 `smooth=1e-5` 防止分母为 0。

```python
class RandomGenerator(object):
    def __init__(self, output_size):
        self.output_size = output_size       # [224, 224]

    def __call__(self, sample):
        image, label = sample['image'], sample['label']

        # 增强策略（二选一）：
        if random.random() > 0.5:
            image, label = random_rot_flip(image, label)    # 50% 概率
        elif random.random() > 0.5:
            image, label = random_rotate(image, label)      # 25% 概率

        # 缩放到固定大小 224×224
        x, y = image.shape
        if x != self.output_size[0] or y != self.output_size[1]:
            image = zoom(image, (224/x, 224/y), order=3)    # 3次样条插值
            label = zoom(label, (224/x, 224/y), order=0)    # 最近邻插值

        image = torch.from_numpy(image.astype(np.float32)).unsqueeze(0)
        label = torch.from_numpy(label.astype(np.float32))
        return {'image': image, 'label': label.long()}
```

**功能说明：**

- 图像使用 3 次样条插值（`order=3`），标签使用最近邻插值（`order=0`）——防止标签值被插值改变

```python
def test_single_volume(image, label, net, classes, patch_size=[256,256], ...):
    # 如果是 3D 体数据
    if len(image.shape) == 3:
        prediction = np.zeros_like(label)
        for ind in range(image.shape[0]):     # 逐切片处理
            slice = image[ind, :, :]
            # 缩放到 224×224
            if slice.shape[0] != patch_size[0] or slice.shape[1] != patch_size[1]:
                slice = zoom(slice, (224/x, 224/y), order=3)
            input = torch.from_numpy(slice).unsqueeze(0).unsqueeze(0).float().cuda()
            # 推理
            with torch.no_grad():
                outputs = net(input)
                out = torch.argmax(torch.softmax(outputs, dim=1), dim=1).squeeze(0)
                out = out.cpu().detach().numpy()
                # 缩放回原始尺寸
                if x != patch_size[0] or y != patch_size[1]:
                    pred = zoom(out, (x/224, y/224), order=0)
                prediction[ind] = pred

    # 计算每个类别的 DSC 和 HD95
    metric_list = []
    for i in range(1, classes):  # 跳过背景类 (class 0)
        metric_list.append(calculate_metric_percase(prediction == i, label == i))
    return metric_list
```

**功能说明：**
1. **逐切片推理**：3D CT 体数据被拆分为独立的 2D 切片，逐一送入模型推理
2. **缩放适配**：切片尺寸不匹配时，使用 `scipy.ndimage.zoom` 缩放到 224×224（图像用 3 次插值，预测用最近邻）
3. **argmax 获取预测类别**：`torch.argmax(torch.softmax(outputs, dim=1), dim=1)` 取每个像素中最大概率的类别
4. **拼接还原**：所有切片预测结果沿深度维度拼接为完整的 3D 预测

### 11.3 指标计算

**文件：** `utils.py` 第 48-58 行，`calculate_metric_percase()` 函数

```python
def calculate_metric_percase(pred, gt):
    pred[pred > 0] = 1     # 二值化
    gt[gt > 0] = 1
    if pred.sum() > 0 and gt.sum() > 0:
        dice = metric.binary.dc(pred, gt)      # medpy 库的 Dice
        hd95 = metric.binary.hd95(pred, gt)    # medpy 库的 HD95
        return dice, hd95
    elif pred.sum() > 0 and gt.sum() == 0:     # 假阳性
        return 1, 0
    else:                                       # 假阴性或无目标
        return 0, 0
```

**功能说明：** 使用 `medpy` 库计算二值分割的 Dice 系数和 95% Hausdorff 距离。边界情况处理：预测有但真值无 → (1,0)；都无 → (0,0)。评估从 class 1 开始（跳过背景 class 0），与论文一致。
